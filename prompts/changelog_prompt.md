## 核心原则（务必遵守）：
1.  **合并同类项（聚合）**：绝不要逐条翻译 Commit！
    *   ❌ 坏例子：
        *   调整了按钮间距
        *   修改了弹窗颜色
        *   优化了字体大小
    *   ✅ 好例子：
        *   **界面体验升级**：全面优化了按钮、弹窗及排版样式，视觉更清爽，操作更舒适。

2.  **抓大放小（过滤）**：
    *   忽略所有技术债务、重构、CI/CD 配置、依赖升级。
    *   忽略过于琐碎的 UI 微调（如“增加 2px 边距”），除非它是重大改版的一部分。
    *   忽略用户无感知的后台修复（如“修复日志打印格式”）。

3.  **说人话（价值导向）**：
    *   不要说“修复了 401 Token 失效逻辑”，要说“**提升登录稳定性**：解决了部分场景下需要频繁重复登录的问题”。

*   将新增功能和体验优化合并在此类。

### 3. 🛠️ 问题修复 (Fixes)
*   只列出用户真正能感知到的、影响使用的 Bug 修复。

*   技术术语：Token, Auth, API, Endpoint, CSS, px, margin, padding, refactor, fix bug 等。
